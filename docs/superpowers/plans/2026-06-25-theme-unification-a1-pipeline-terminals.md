# Theme Unification A1 — Pipeline + Terminal/TTY Slice Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the wana→dotfiles theme pipeline (generate → vendor → switch) and wire it end-to-end for kitty, alacritty, and the Linux TTY console, proving the architecture before fanning out to the remaining targets.

**Architecture:** wana (`tools/gen.py`) renders per-app theme files from the base24 schemes. A dotfiles `theme-sync` script runs that generator and vendors the outputs into module paths (committed). A single declarative `theme-manifest` maps each target to its source/dest/symlink/reload. A runtime `theme-switch {dark|light|toggle}` repoints per-app `current` symlinks and nudges live apps. No runtime dependency on wana or Python.

**Tech Stack:** Python 3 (wana generator, `str.format` templates, `unittest`), POSIX sh (dotfiles scripts), GNU stow (existing `bootstrap.sh` deploy).

**Scope note:** This is plan **A1** of sub-project A. Out of scope here (own follow-up plans, same recipe): A2 = bat/delta/fzf/starship/btop/yazi/lazygit/opencode/atuin/herdr/pywal-scheme; A3 = noctalia. Spec: `docs/superpowers/specs/2026-06-25-theme-unification-wana-design.md`.

## Manifest format (reference — created in Task 3)

Pipe-delimited, one row per target. `{v}` = variant (`dark`/`light`). Home paths are `$HOME`-relative (no leading slash). Reload tokens: `{cur}` = absolute current-symlink path, `{var}` = absolute resolved variant path.

```
# name | wana_src{v} | repo_dest{v} | home_current | home_variant{v} | reload
```

def render_tty(s: Scheme) -> tuple[str, str]:

```sh
#!/usr/bin/env sh
# theme-sync — regenerate wana themes and vendor them into this repo.
#   theme-sync          # regenerate (runs wana gen.py) and copy per manifest
#   theme-sync --check  # exit 1 if any vendored file is stale (no writes)
# Requires wana checked out (dev-time only). Override its path with WANA_DIR.
set -eu

DOTFILES="$(cd "$(dirname "$0")/.." && pwd)"
WANA="${WANA_DIR:-$HOME/Repos/wana}"
MANIFEST="$DOTFILES/script/theme-manifest"
CHECK=0
[ "${1:-}" = "--check" ] && CHECK=1

[ -d "$WANA" ] || { echo "wana not found at $WANA (set WANA_DIR)"; exit 2; }

if [ "$CHECK" -eq 1 ]; then
  python3 "$WANA/tools/gen.py" --check
else
  python3 "$WANA/tools/gen.py"
fi

stale=0
while IFS='|' read -r name src dest _cur _var _reload; do
  case "$name" in ''|\#*) continue ;; esac
  for v in dark light; do
    s="$WANA/$(echo "$src"  | sed "s/{v}/$v/g")"
    d="$DOTFILES/$(echo "$dest" | sed "s/{v}/$v/g")"
    [ -f "$s" ] || { echo "MISSING wana output: $s"; exit 3; }
    if [ "$CHECK" -eq 1 ]; then
      if ! cmp -s "$s" "$d"; then echo "STALE: $d"; stale=1; fi
    else
      mkdir -p "$(dirname "$d")"
      cp "$s" "$d"
      echo "vendored $name/$v -> ${d#"$DOTFILES"/}"
    fi
  done
done < "$MANIFEST"

[ "$CHECK" -eq 1 ] && [ "$stale" -eq 1 ] && { echo "Run: script/theme-sync"; exit 1; }
[ "$CHECK" -eq 1 ] && echo "OK: vendored themes up to date"
exit 0
```

### Task 5: dotfiles — theme-switch (runtime switcher)

**Files:**

- Create: `config/misc/scripts/.local/script/theme-switch`

- [ ] **Step 1: Write `theme-switch`**

Create `config/misc/scripts/.local/script/theme-switch` (make executable):

```sh
#!/usr/bin/env sh
# theme-switch — apply the wana palette across apps by repointing per-app
# `current` symlinks and nudging live-reloadable apps.
#   theme-switch dark|light   # set variant
#   theme-switch toggle       # flip current variant (default dark)
#   theme-switch apply        # re-apply the saved variant (e.g. on login)
# Reads $HOME-relative paths from the repo manifest; safe headless/server.
set -eu

DOTFILES="${DOTFILES_DIR:-$HOME/Repos/dotfiles}"
MANIFEST="$DOTFILES/script/theme-manifest"
STATE="$HOME/.cache/wana/variant"

current_variant() { [ -r "$STATE" ] && cat "$STATE" || echo dark; }

case "${1:-}" in
  dark|light) variant="$1" ;;
  toggle)     [ "$(current_variant)" = dark ] && variant=light || variant=dark ;;
  apply|'')   variant="$(current_variant)" ;;
  *) echo "usage: theme-switch dark|light|toggle|apply"; exit 64 ;;
esac

[ -r "$MANIFEST" ] || { echo "manifest not found: $MANIFEST"; exit 2; }

mkdir -p "$(dirname "$STATE")"
printf '%s\n' "$variant" > "$STATE"

deferred=""
while IFS='|' read -r name _src _dest home_current home_variant reload; do
  case "$name" in ''|\#*) continue ;; esac
  cur="$HOME/$home_current"
  var="$HOME/$(echo "$home_variant" | sed "s/{v}/$variant/g")"
  if [ ! -e "$var" ]; then
    echo "skip $name: $var missing (run setup.sh use, then theme-sync)"
    continue
  fi
  mkdir -p "$(dirname "$cur")"
  ln -sfn "$var" "$cur"
  cmd="$(echo "$reload" | sed -e "s|{cur}|$cur|g" -e "s|{var}|$var|g")"
  if eval "$cmd" >/dev/null 2>&1; then
    :
  else
    deferred="$deferred $name"
  fi
done < "$MANIFEST"

echo "theme: $variant"
[ -n "$deferred" ] && echo "next-launch only (not live-reloaded):$deferred"
exit 0
```

## How to Add a Theme Target

The wana palette drives app theming through a generate → vendor → switch pipeline.

1. **wana repo:** add `render_<app>()` + `tools/templates/<app>` in `wana/tools/gen.py`; extend `tools/test_gen.py`; run `python3 tools/gen.py` and commit the generated `themes/<app>/`.
