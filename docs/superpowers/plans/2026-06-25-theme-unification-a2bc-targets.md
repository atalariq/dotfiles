# Theme Unification A2bc ÔÇö starship/btop/lazygit/opencode + Noctalia cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring starship, btop, lazygit, and opencode under the wana palette, and stop Noctalia's templating from competing for the apps the wana pipeline now owns (disable its alacritty/btop/kitty/starship templates + remove the artifacts they left).

**Architecture:** Reuses the A1/A2a pipeline. starship and lazygit are **in-place single-variant** edits (their themes live inside an existing config, pasted between markers). btop is a **Class A symlink** target (dark/light switch). opencode is a **single-file** theme (its own dark/light defs in one JSON), vendored sync-only.

**Tech Stack:** Python 3 (wana `gen.py`, `str.format` templates, `unittest`), POSIX sh, GNU stow.

**Scope:** A2bc = A2b + A2c minus yazi (skipped per user) and minus atuin (dropped: atuin themes accept only _named_ palette colors, not hex, so they can't represent the wana palette) and minus herdr (deferred to sub-project B; its `config.toml` is actively hand-edited). Spec: `docs/superpowers/specs/2026-06-25-theme-unification-a2-cli-targets-design.md`. Builds on A1+A2a (merged to main).

**Repo paths:** `WANA` = `~/Repos/wana`, `DOTFILES` = `~/Repos/dotfiles`.

**Branches:** `cd ~/Repos/wana && git checkout -b feat/theme-renderers-a2bc`; `cd ~/Repos/dotfiles && git checkout -b feat/theme-a2bc-targets`.

**wana base24 slots (dark scheme) for reference** (renderers map roles to these):
base00 bg, base01 surface, base02 overlay/sel, base03 muted, base04 dark-fg, base05 text, base06 light-fg, base07 lightest, base08 red, base09 orange, base0A yellow, base0B green(accent), base0C cyan, base0D blue, base0E magenta, base0F brown, base10 darker-bg, base11 darkest-bg.
